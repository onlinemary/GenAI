# 1D Generative Adversarial Network (GAN) for Quadratic Function Approximation

This repository contains a simple, clean implementation of a Generative Adversarial Network (GAN) built from scratch using Keras and NumPy. The objective of the network is to learn how to generate data points that follow a simple 1D quadratic function ($y = x^2$).

It serves as an excellent foundational project for understanding how Generator and Discriminator networks interact during the adversarial training process.

---

## 📌 Project Overview

A GAN consists of two neural networks competing against each other in a zero-sum game:

1. **The Generator:** Tries to create realistic data points (in this case, points lying on a parabola $y = x^2$) from random noise (latent space).
2. **The Discriminator:** Tries to distinguish between real data points (from the true dataset) and fake data points (created by the generator).

As training progresses, the Generator gets better at faking the data, while the Discriminator gets better at catching the fakes, eventually driving the Generator to accurately reproduce the target function geometry.

---

## 🛠️ Features

* **Custom Data Generation:** Generates real data points following $y = x^2$ for $x \in [-0.5, 0.5]$.
* **Standalone Models:** Modular design separating the Generator, Discriminator, and the combined GAN model structures.
* **Manual Training Loop:** Implements manual batch updates (`train_on_batch`) to alternate optimization between the Discriminator and Generator.
* **Live Performance Monitoring:** Periodically evaluates accuracy and visualizes the generated data points against real data using `matplotlib` scatter plots.

## ⚙️ How It Works

### Model Architecture

* **Generator:** Accepts a 5-dimensional random latent vector, passes it through a dense layer (20 neurons, ReLU), and outputs a 2D coordinate $(x, y)$.
* **Discriminator:** Accepts a 2D coordinate, passes it through a hidden dense layer (25 neurons, ReLU), and outputs a probability score using a Sigmoid activation (1 = Real, 0 = Fake).

### Training Setup

The training loop alternates between two phases every epoch:

1. **Train Discriminator:** Fed with a half-batch of real samples (labeled as `1.0`) and a half-batch of fake samples (labeled as `0.0`).
2. **Train Generator:** Feeds random latent points into the GAN model with target labels inverted to `1.0`. Because the discriminator's weights are frozen in this step, the backpropagation updates only the generator to make its outputs look more "real".

---

## 🚀 Usage

You can run the script directly from your IDE (e.g. VS Code).


### Configuration Parameters

You can tweak the training dynamics by adjusting the variables at the bottom of the script:

* `latent_dim`: Dimension of the random noise input space (Default: `5`).
* `n_epochs`: Total training iterations (Default: `30000`).
* `n_batch`: Batch size for updates (Default: `128`).
* `n_eval`: How often to print accuracy results and generate scatter plots (Default: `200`).

---

## 📊 Evaluation & Output

Every `200` epochs, the script will output the accuracy of the discriminator on both real and fake batches:

```text
8599 0.384 0.620
33599 0.490 0.460
...

```

It will also trigger a `matplotlib` pop-up window visualizing the distribution:

* **Red Points:** Real data sample ($y = x^2$).
* **Blue Points:** Fake data generated by your network.

Over time, you will watch the blue points shift from a random cloud of noise into the distinct parabolic shape of the red points.
